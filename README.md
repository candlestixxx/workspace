# Workspace (Monorepo)

This workspace serves as a centralized monorepo and orchestration hub for various independent projects. Each project is managed separately in its own session, while this root context handles cross-project organization, documentation, and synchronization.

## Workspace Structure
- **Docs**: Centralized documentation in `docs/`. Root-level documentation is kept lean, with historical plans and analyses archived in `docs/archive/`.
- **Scripts**: Utility scripts for workspace management in `scripts/`.
- **Logs**: Operation logs and session handoffs in `logs/`.
- **Playground**: Temporary or experimental files in `playground/`.

## Key Projects
- **aios**: AI Operating System & Agent Infrastructure.
- **fwber**: Dating platform (Laravel/Next.js/PostgreSQL).
- **Hellven**: Unity game development.
- **itgmania**: Rhythm game engine (StepMania fork).
- **okgame**: Cross-platform game engine.
- **filez**: Desktop file management tool.
- **ArrowVortex**: Rhythm game editor.

## Development Philosophy
- **Independent Contexts**: Each subproject has its own documentation, goals, and tasks.
- **AI Orchestration**: Multi-model coordination for rapid development and high-quality results.
- **Flat Organization**: Projects are maintained as top-level directories or submodules for direct access.
